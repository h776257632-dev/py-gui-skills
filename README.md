# 🏗️ AI-CTO：架构文档自动同步技能

> 让 AI 在每次写完代码后，自动生成或更新一份完整的《系统架构与工程文档》。

---

## 📌 这是什么？

这是一个 **AI 技能（Skill）**，安装在你的工作区后，AI 在帮你写完代码后会被你手动触发，
自动扫描项目代码，然后逆向梳理出一份完整的 `ARCHITECTURE.md` 架构文档。

**核心原则：全文覆盖** — 即使只改了一个小 Bug，输出的也是完整版本，直接覆盖旧文档即可。

---

## 🗂️ 文件结构

```
ai-cto/
├── README.md                                          ← 你正在看的说明文档
└── .agent/
    └── skills/
        └── architecture-sync/
            └── SKILL.md                               ← AI 技能指令文件 (系统必须路径)
```

> **为什么 SKILL.md 要放在 `.agent/skills/` 下？**
> 这是 AI 系统自动发现技能的约定路径，类似于 `.vscode/` 或 `.github/` 的约定。
> 放在这里 AI 才能自动识别和读取这个技能。

---

## 🚀 如何使用

### 安装

将本项目 (`ai-cto/`) 文件夹放在你的工作区根目录下即可（或者在多项目技能目录中统一管理）。

### 触发方式

在 AI 帮你写完代码之后，用以下任意表述触发：

```
调用架构文档同步
执行架构文档同步
同步架构文档
更新 ARCHITECTURE.md
```

### 产出物

AI 会在 **你当前项目的根目录** 生成或更新一个文件：

```
你的项目/
├── ARCHITECTURE.md    ← AI 自动生成的架构文档
├── src/
└── ...
```

## 📋 文档覆盖 9 大板块

| # | 板块名称 | 内容 | 是否必填 |
|---|----------|------|----------|
| 1 | **变更记录** | 版本号、变更类型、核心变更摘要 | ✅ 必填 |
| 2 | **核心架构拓扑** | 分层结构表 + Mermaid 架构图 + 模块交互 | ✅ 必填 |
| 3 | **通信协议与接口** | BLE/串口帧格式、指令字典、HTTP API 等 | ✅ 必填 |
| 4 | **软件/固件设计** | 状态机、并发同步、持久化存储 | ✅ 必填 |
| 5 | **项目文件结构** | 目录树 + 每个文件/目录的职责标注 | ✅ 必填 |
| 6 | **硬件资源映射** | 引脚分配表、外设占用 | ⚙️ 若无可省略 |
| 7 | **依赖与环境** | 依赖库清单 + 构建/编译说明 | ✅ 必填 |
| 8 | **关键设计决策** | 技术选型及其理由 | ✅ 必填 |
| 9 | **已知问题与路线图** | Bug 追踪 + 未来规划 | ⚙️ 若无可省略 |

└─────────────┘     └──────────────────────┘     └────────┬────────────┘
                                                          │
                                                          ▼

## 💡 适用场景

- **嵌入式 / IoT 项目**：BLE 通信协议、FreeRTOS 多任务、硬件引脚映射
- **前后端 Web 项目**：HTTP/WebSocket API、状态管理、组件结构
- **微信小程序**：页面路由、数据流、BLE 通信
- **混合项目**：以上任意组合

---

## ⚠️ 注意事项

1. AI 只会根据 **代码实际内容** 来撰写文档，不会凭空想象功能。
2. 如果项目中没有通信协议，"通信协议"板块会标注"暂无"。
3. 每次都是全文覆盖，历史变更用文档开头的 Changelog 表格追踪。
4. 版本号自动递增（v1.0 → v1.1 → v1.2 → ...）。
